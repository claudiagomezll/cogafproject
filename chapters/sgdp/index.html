Chapter Three

Design Pattern for Serious Games

This chapter presents the use of four design patterns for SG based on the cognitive-affective model (COGAF) to reduce the complexity of developing SGs for cognitive-affective training, providing clean code structures that are easy to understand and adaptable to any programming language. The proposal of this article starts with the specification of the COGAF model needs, and then proceeds to identify the design patterns that provide a solution to each need. The SDPs used in the recurring scenarios of the COGAF model for SG create a set of Java classes. These classes embody the architectural principles of the SGDPs for cognitive-affective training. The SGDPs are validated using an illustrative example for the production of an SG based on the COGAF model. The SG result is tested firstly with the user and secondly using competency questions. The validation results show that from the preliminary code generated based on the COGAF model extended to design patterns, a SG prototype was achieved that meets completeness and consistency criteria with respect to the model used
